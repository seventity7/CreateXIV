### 🔧 Dalamud Installation

| Step | Action |
| --- | --- |
| 1️⃣ | Open the Dalamud plugin list with `/xlplugins` |
| 2️⃣ | Search for `CreateXIV` |
| 3️⃣ | Click **Install** |
| 4️⃣ | Enable the plugin |
| ✔ | Open the main window or type `/create` |

</div>

> [!TIP]
> CreateXIV is intended to be installed through the ingame Dalamud plugin installer only.

### 🧠 How It Works

### 🪟 Main Window

<img width="1094" height="585" alt="image" src="https://github.com/user-attachments/assets/c2cfe5d5-f1ab-43e1-b64b-34147684b315" />

CreateXIV allows you to define aliases that point to commands or macros.

---

### 🔹 Alias → Command

> [!NOTE]
> Use this when you want a shorter custom command to execute a longer command.

**Example:**
| Alias | Command |
| --- | --- |
| `Golem` | `/Example Dynamis Golem` |

Typing this in chat:
```txt
/Golem
```

Executes:
```txt
/Example Dynamis Golem
```

---

### 🔹 Alias → Macro

> [!NOTE]
> Use this when you want an alias to execute a macro from your in-game macro list.

**Example:**
| Alias | Macro |
| --- | --- |
| `Golem` | `macro:45` |

Typing this in chat:
```txt
/Golem
```

Executes:
```txt
Macro 45
```

---

### 🔹 Categories/Favorites

While creating command or macro aliases, you can create new categories or use existing ones. Categories can have their own color.
You can also pin/favorite an alias to keep it at the top of the list.

<a id="important-rules"></a>
<div align="center">

### ⚠️ Important Rules

| Rule | Status |
| --- | --- |
| Do **not** use `/` in alias names | ❌ Invalid |
| Macro/Command chains are not supported | ❌ Unavailable |
| Always use `/` in the command field for plugin commands | ✅ Required |
| You can't create an alias named `create` | ❌ Reserved |
| You can't point an alias command to `/create` | ❌ Reserved |
| Duplicate alias names overwrite existing entries | 🔁 Expected |
| The macro must exist in-game | ✅ Required |
| Macro content is managed inside FFXIV, not inside the plugin | ℹ️ Note |

</div>

> [!IMPORTANT]
> These rules help prevent command conflicts and invalid alias behavior.
